Vision Summary
What We're Building
A compatibility layer,
not just a booking app.
Hair Harmony matches clients to stylists based on real compatibility — hair type, budget, vibe, and style — not just proximity and availability. We're validating that curated matching improves booking confidence and satisfaction.
Product Documentation
Product Spec
MVP feature set, user roles, and matching logic. Use this as the canonical reference for what's in scope.
User Roles
👤
Client
Posts hair requests · Reviews stylist matches · Accepts or declines · Chats before booking · Books after match.
✂️
Stylist
Creates specialization profile · Browses compatible requests · Accepts or passes · Chats · Confirms booking.
Must Have — MVP
Authentication MVP
Email + password sign up, role selection (Client / Stylist), profile completion required, secure sessions, password reset.
Profile Setup (Both Roles) MVP
Clients define hair type, budget, vibe. Stylists define specialties, excluded styles, price range, availability, vibe style.
Create Request + Request Card MVP
Client submits hair type, desired style, reference photo, budget range, timeframe, location, vibe preference. Creates a Request Card with status: Open / Matched / Closed / Booked.
Stylist Accept / Pass System MVP
Stylists see Request Cards filtered by compatibility. Can accept or pass. Accepted requests move to "Pending Mutual."
Mutual Match Logic MVP
Rule-based: hair type, style, budget range, location radius, availability. Vibe alignment as optional boost. No AI needed for MVP.
Messaging (Post-Match Only) MVP
Unlocked only after mutual acceptance. One-to-one chat, image sharing, message history.
Basic Booking MVP
Client selects from stylist availability, confirms booking. States: Requested / Confirmed / Completed / Cancelled. Deposit optional.
Can Wait — Post-MVP
AI Matching Post-MVP
Intelligent pairing beyond rule-based filtering. Learns from booking outcomes and satisfaction scores.
Notifications (Email + Push) Post-MVP
New request, mutual match, new message, booking confirmed, booking reminder triggers.
Admin Panel Post-MVP
Approve stylists, remove inappropriate requests, monitor reports, view match statistics and booking conversion rates.
Ratings & Reviews Post-MVP
Post-booking ratings for clients (style, communication, satisfaction) and stylists (client communication, professionalism).
Subscription Tiers, Dispute Resolution, Analytics Post-MVP
Monetization layer, advanced dispute automation, and business intelligence dashboards.

Usage Rules
✓ Do
  • Use Warm Cream as default page background
  • Terracotta for primary CTAs only
  • Pair Cormorant Garamond with DM Sans
  • Use Espresso dark sections for visual rhythm
  • Keep buttons pill-shaped (border-radius: 50px)
  • Use italic Cormorant for editorial moments
✗ Don't
  • Terracotta on large background areas
  • Pure white (#FFFFFF) — always use Warm White
  • DM Sans for display headlines
  • More than 2 accent colors in one section
  • Black (#000000) — use Espresso instead
  • Square or sharply cornered buttons
